知序多端前端设计方案稿 v2.0

产品定位：AI 学习科研管家 / 个人学习科研 Agent OS
核心原则：对话驱动、任务中台、主动陪跑、交互共创、证据可查、跨项目沉淀、多端连续
适用端：Web 桌面端、移动端、平板端、PWA/桌面端、组织版工作台
设计结论：知序不能被设计成单项目工作台，也不能被设计成聊天工具、项目管理后台或文档生成器。知序的前端应该是一个“AI 对话即首页”的个人学习科研操作系统：用户从任何端进入产品，第一眼都应看到 AI 对话主界面。今日、任务、项目、产物、知识、合规、Skills、设置等能力都必须围绕 AI 对话页展开，由 Agent 在对话中显式理解、规划、调用、执行、推送和沉淀。

?

1. 总体判断：上一版为什么格局小

上一版方案虽然纠正了“项目管理大盘”的问题，但仍然把知序理解成“某个项目里的对话 + Canvas”。这只能覆盖你最初想法中的一个子场景：老师布置 PPT，用户上传文件，AI 生成选题并共创产物。

但完整 PRD 的范围远大于此。知序真正要解决的是：本科生、硕士生、博士生、科研人员每天面对大量零散任务、多个交付物、多种 AI 工具、多端设备、多份资料、多类截止时间和长期知识资产时，缺少一个统一的个人中台。

因此，前端设计必须先确立一个最高优先级原则：所有端口进入后的首页都是 AI 对话页。不同端的区别不是“首页不同”，而是 AI 对话页承载的信息密度、展开方式和可执行操作不同。

前端设计必须从“单项目工作台”升级为“五层操作系统”：
	1.	捕获层：手机、网页、桌面、平板随时收任务、收资料、收灵感。
	2.	编排层：Agent 把任务拆成项目、计划、依赖、确认节点和风险。
	3.	共创层：用户和 AI 在文档、PPT、论文、题库、实验日志中逐步共创。
	4.	监督层：三色溯源、引用核验、Human Gate、版本差异、合规报告。
	5.	沉淀层：知识胶囊、导师偏好、错题归因、实验复盘、跨项目复用。

知序网页端的首页不应只展示“一个项目”，而应展示“我的学习科研系统现在处于什么状态”。

2. 产品前端总架构：AI 对话首页 + 五大空间

2.0 最高层首页原则

知序所有端口的默认首页必须是 AI 对话页面，而不是 Dashboard、项目列表、功能宫格或资料库。

原因：用户购买知序的核心预期不是“管理项目”，而是“把任务交给 AI，让 AI 主动理解、拆解、执行、提醒、核验和沉淀”。因此，用户每次打开产品时，最应该看到的是一个正在工作的 AI，而不是一堆需要自己理解的功能入口。

各端首页统一定义为：AI Conversation Home。

它必须包含：
	1.	AI 今日简报：告诉用户 AI 已经处理了什么、发现了什么风险、现在等用户确认什么。
	2.	任务型对话流：用户可以直接输入任务，也可以继续上一个任务。
	3.	显式工作卡片：AI 理解卡、计划卡、Skill 调用卡、执行进度卡、产物卡、风险卡、Human Gate 卡。
	4.	待确认入口：选题、大纲、引用、导出、敏感资料上云等确认动作。
	5.	后台处理状态：文件解析、文献矩阵、PPT 生成、引用核验、复习计划重排等。
	6.	快捷动作：上传资料、语音任务、继续推进、安排今天、查看产物、进入 Canvas。
	7.	可展开空间入口：今日、任务、项目、产物、知识、合规、Skills 都作为 AI 对话页的侧栏、抽屉、卡片或 Command Menu 出现。

禁止首页形态：
	1.	以项目统计为首页。
	2.	以功能九宫格为首页。
	3.	以空白聊天框为首页。
	4.	以资料库为首页。
	5.	以知识图谱为首页。
	6.	以普通 Todo List 为首页。

五大空间仍然存在，但它们不作为独立首页抢占入口，而是从 AI 对话首页中被 Agent 显式调用、展开和推送。

知序前端应由五个一等空间组成，而不是简单功能菜单。

2.1 今日空间 Today Command Center

定位：用户每天打开知序的第一站。
核心问题：今天我该处理什么？AI 已经帮我准备好了什么？哪些任务有风险？

今日空间是全局指挥中心，不是统计 Dashboard。它聚合课程、科研、考试、小组、实验、导师反馈、投稿和知识胶囊推荐。

必须展示：
	1.	今日待确认：AI 已完成预处理，等待用户选题、确认大纲、核验引用、选择方案。
	2.	今日待执行：用户必须亲自完成的任务，例如阅读、做题、实验、确认结论。
	3.	AI 后台进度：正在解析资料、生成 PPT 大纲、核验引用、整理导师反馈。
	4.	截止风险：即将到期的 PPT、报告、考试、组会、返修、投稿材料。
	5.	主动建议：系统根据时间、任务状态和用户习惯给出的下一步建议。
	6.	跨项目提醒：某个旧项目的知识胶囊可用于当前任务。

页面第一句不应是“欢迎回来”，而应是类似：

“今天有 4 件事需要你确认：机器学习 PPT 选题、统计学复习计划、导师反馈整改、组会汇报大纲。AI 已经处理完 12 个文件。”

2.2 任务空间 Task Inbox / Mission Layer

定位：所有学习科研任务的统一收件箱。
核心问题：我所有杂乱任务都在哪里？每个任务现在走到哪一步？

任务空间不是传统 Todo List。每个任务都是一个带状态机的 Agent 项目。

任务类型：
	1.	课程作业。
	2.	PPT / Presentation。
	3.	Word / 报告。
	4.	论文精读。
	5.	文献综述。
	6.	期末复习。
	7.	小组作业。
	8.	实验任务。
	9.	组会汇报。
	10.	导师反馈整改。
	11.	审稿意见返修。
	12.	基金/课题申报。
	13.	学术简历。
	14.	竞赛/评优材料。

每个任务卡必须显示：
	1.	任务目标。
	2.	当前状态：已捕获、理解中、待确认、执行中、核验中、可导出、已归档。
	3.	AI 已完成事项。
	4.	用户下一步。
	5.	截止时间。
	6.	风险等级。
	7.	产物入口。
	8.	资料数量。
	9.	三色溯源状态。
	10.	是否可复用历史胶囊。

2.3 项目空间 Project Workspace

定位：单个任务/项目的深度推进场所。
核心问题：这个任务怎么一步步做完？

项目空间不是项目管理后台，而是对话驱动的任务推进现场。

项目空间默认三栏：
	1.	左栏：项目状态机、任务阶段、产物列表、历史确认。
	2.	中栏：任务型对话流、选题卡、方案卡、大纲卡、AI 进度卡。
	3.	右栏：资料、老师/导师要求、截止日期、三色溯源、待核查项、活动记录。

项目空间必须支持模式切换：
	1.	规划模式：看任务拆解、三方案、今日计划。
	2.	资料模式：看文件、摘要、文献矩阵、课堂纪要。
	3.	共创模式：进入 PPT/文档/题库/实验日志 Canvas。
	4.	核验模式：看引用、三色溯源、合规风险。
	5.	复盘模式：生成知识胶囊和下一次复用建议。

2.4 产物空间 Artifact Studio

定位：真正完成 PPT、报告、综述、复习计划、题库、实验日志、讲稿、简历、申报材料的地方。
核心问题：我能不能深度参与每一页、每一段、每一个引用，而不是被 AI 一键生成绑架？

Artifact Studio 是知序前端最重要的生产力界面。它必须统一承载所有产物类型。

布局：
	1.	左侧 AI 控制台：用户用自然语言要求修改、补充、核验、压缩、转格式。
	2.	中央 Canvas：PPT 页面、Word 文档、论文矩阵、复习计划、实验记录等。
	3.	右侧 Inspector：当前块的目标、证据、三色溯源、评论、版本、质量问题。

核心能力：
	1.	Block Editing：所有内容按块编辑。
	2.	Inline AI Command：选中段落或页面后局部指令。
	3.	Evidence Binding：每个观点绑定来源。
	4.	Version Diff：每次 AI 或用户修改可对比。
	5.	Comment Layer：导师反馈、小组评论、用户自评可落在具体块上。
	6.	Export Check：导出前检查引用、格式、风险和用户确认。

2.5 知识空间 Knowledge OS

定位：长期知识资产与复用系统。
核心问题：我做过的课程、论文、实验、错题和反馈能不能变成下一次任务的起点？

知识空间不是普通资料库。它包括：
	1.	知识胶囊。
	2.	课程知识图谱。
	3.	错题归因库。
	4.	文献矩阵库。
	5.	实验日志库。
	6.	导师偏好库。
	7.	术语统一库。
	8.	个人学术履历资产。
	9.	同门共享知识库。
	10.	Skill 能力市场。

知识空间必须支持跨项目推荐。例如用户新建一个机器学习 PPT，系统应主动提示：

“你上学期做过 Transformer 课程报告，其中 3 个知识点和当前任务相关。是否复用大纲结构和术语表？”

3. Web 桌面端总体布局

3.0 核心修正：AI 对话页是所有能力的主操作面

知序所有能力都必须存在，但不应让用户通过复杂导航主动寻找。最好的前端形态是：AI 对话页作为统一主入口，其他空间作为对话中的显式工作面板被调用、展开、执行和推送。

这意味着，用户不是进入“PPT 功能”“论文功能”“实验功能”“复习功能”，而是在 AI 对话页里说出任务，系统根据上下文显式调用对应能力：
	1.	需要做 PPT：对话中出现选题卡、大纲卡、PPT Canvas、页级 Inspector。
	2.	需要读论文：对话中出现文献矩阵、研究空白卡、组会汇报角度。
	3.	需要复习考试：对话中出现考试倒计时、三方案复习计划、题库、错题归因。
	4.	需要做实验：对话中出现实验日志、参数版本、异常归因、下次实验计划。
	5.	需要处理导师反馈：对话中出现反馈拆解表、整改追踪、导师偏好沉淀。
	6.	需要合规核验：对话中出现三色溯源面板、引用核验报告、Human Gate。
	7.	需要沉淀复用：对话中出现知识胶囊草稿、历史项目推荐、跨项目复用建议。

AI 对话页不是普通聊天页，而是“可视化 Agent 工作台”。它必须显式展示 Agent 的工作过程：理解了什么、计划怎么做、调用了什么能力、正在执行什么、哪些结果已完成、哪些地方需要用户确认、后续会主动推送什么。

重要边界：前端展示的是可审计、可解释的工作过程，也展示模型私密推理链，并且向现在流行的agent工具一样，有工具调用的展示和流畅自然高级的动效。用户看到的是任务理解、计划、证据、工具调用、执行状态、风险判断和确认节点。

3.1 Web 端首页布局

Web 端进入后的首页必须是 AI Conversation Home。

Web 首页默认结构：
	1.	左侧任务流：最近任务、待确认任务、AI 正在处理、即将截止。
	2.	中央 AI 对话流：AI 今日简报、任务对话、工作卡片、确认卡、产物预览。
	3.	右侧上下文面板：资料、证据、日程、活动、合规风险，随当前对话动态变化。
	4.	底部命令栏：自然语言输入、上传、语音、快捷命令、当前模式。

用户的典型路径不是“打开项目列表 → 点某个项目 → 看数据”，也不是“打开今日页 → 再找功能”，而是：

“打开知序 → AI 对话页直接告诉我已处理什么、待确认什么、今天该做什么 → 我在对话中确认选题/方案/大纲 → 对话中展开 Artifact Studio → 深度共创 → 导出 → 生成知识胶囊。”

3.2 全局框架

Web 端采用“轻导航 + 主工作区 + 智能上下文”的框架。

全局左侧导航：
	1.	今日。
	2.	任务。
	3.	项目。
	4.	产物。
	5.	知识。
	6.	合规。
	7.	Skills。
	8.	设置。

顶部栏：
	1.	全局搜索：搜索任务、资料、产物、引用、知识胶囊。
	2.	快速捕获：新任务、上传资料、语音任务、截图识别。
	3.	AI 状态：后台任务数量、正在处理、失败任务。
	4.	通知：待确认、截止风险、协作评论。
	5.	账号与隐私模式。

右侧全局可折叠面板：
	1.	当前上下文。
	2.	AI 后台队列。
	3.	今日提醒。
	4.	最近活动。
	5.	额度与隐私状态。

4. 核心页面一：AI 对话控制台 AI Conversation Console

4.1 页面目标

AI 对话控制台是知序的主产品页面。用户从这里捕获任务、查看 AI 后台处理、确认关键节点、调用产物工作台、接收主动推送。它不是 ChatGPT 式空白聊天框，也不是消息流，而是一个带显式工作状态、工具调用、任务卡片和产物面板的 Agent Console。

这个页面必须让用户始终知道：
	1.	AI 当前理解的任务是什么。
	2.	AI 正在调用哪些 Skills。
	3.	哪些步骤已经完成。
	4.	哪些结果可查看。
	5.	哪些节点需要用户确认。
	6.	下一步会推进什么。
	7.	有哪些风险、截止日期和待核验内容。

4.2 页面总体布局

桌面端推荐四区布局：
	1.	左侧：任务收件箱与项目流。
	2.	中央：AI 对话与显式工作流。
	3.	右侧：上下文与证据面板。
	4.	底部：任务命令输入栏。

其中中央区是核心。所有空间能力都可以在中央对话流中以卡片、面板、时间线、Canvas 快照、任务表、调用日志的形式出现。

4.3 中央对话流结构

中央对话流必须支持以下消息类型：
	1.	用户消息：自然语言、语音转写、文件上传说明、截图说明。
	2.	AI 理解卡：系统结构化展示任务类型、交付物、截止日期、资料范围、缺失信息。
	3.	Agent Plan 卡：展示阶段计划、三方案、依赖关系、责任方、确认节点。
	4.	Skill 调用卡：展示即将调用或正在调用的能力，例如 PDF 解析、OCR、Literature Skill、PPT Skill、Citation Skill。
	5.	执行进度卡：展示后台处理进度，例如“正在解析 6 个文件 4/6”“正在生成 3 个选题”。
	6.	结果卡：展示选题、文献矩阵、复习计划、实验归因、导师反馈拆解等结构化结果。
	7.	Human Gate 卡：高风险确认，例如导出、引用补全、敏感资料上云。
	8.	风险卡：截止风险、资料缺口、灰色内容、引用错误、成本超额。
	9.	Artifact 卡：嵌入 PPT/文档/题库/实验日志的预览，并提供进入 Canvas 的按钮。
	10.	推送卡：系统主动提醒、任务停滞补救、计划重排、知识胶囊推荐。

4.4 Agent 显式工作过程

AI 对话页中必须显式展示以下五类过程，但不能展示模型私密推理链：

A. 理解中
展示系统正在解析用户输入：

“我正在识别这是不是 PPT、报告、考试复习或科研任务。”
“我识别到 3 个交付物：PPT、讲稿、答辩准备。”
“截止日期可能是下周三，需要你确认。”

B. 规划中
展示计划，展示隐藏推理，工具调用的高级动画：

“我会先解析资料，再给你 3 个选题，然后生成页级大纲，最后进入 PPT Canvas。”

Plan 卡字段：
	1.	阶段。
	2.	每阶段目标。
	3.	AI 自动执行部分。
	4.	用户确认部分。
	5.	预计耗时。
	6.	风险。

C. 调用中
Skill 调用必须可见：

“正在调用 PDF Skill 解析 4 个文件。”
“正在调用 Literature Skill 提取论文贡献与局限。”
“正在调用 Citation Skill 核验 12 条引用。”
“正在调用 PPT Skill 生成页级大纲。”

Skill 调用卡字段：
	1.	Skill 名称。
	2.	用途。
	3.	输入范围。
	4.	权限范围。
	5.	执行状态。
	6.	成本/额度。
	7.	是否需要用户确认。

D. 执行中
展示任务进度：
	1.	解析资料。
	2.	生成摘要。
	3.	生成选题。
	4.	生成大纲。
	5.	生成产物草稿。
	6.	核验引用。
	7.	检查风险。
	8.	等待用户。

执行进度不能只是转圈，应展示具体步骤和可见中间结果和工具调用的高级动画。

E. 推送中
Watcher 主动推送：

“你已经 2 天没有推进这个组会项目。我把计划压缩成了 3 个 20 分钟任务。”
“统计学考试还有 9 天，系统已根据错题重新排序今天的复习内容。”
“导师反馈中还有 2 条未整改，建议今天先处理方法部分。”

4.5 对话中的功能显式化

所有功能都可以在对话中以可操作对象出现：
	1.	选题不是纯文本，而是 ChoiceCard。
	2.	计划不是纯文本，而是 PlanOptionCard。
	3.	PPT 不是附件，而是 ArtifactPreview + 进入 PPT Studio。
	4.	文献综述不是长回答，而是 LiteratureMatrixCard。
	5.	复习计划不是列表，而是 ExamPlanCard。
	6.	错题不是答案，而是 MistakeInsightCard。
	7.	实验异常不是猜测，而是 ExperimentHypothesisCard。
	8.	导师反馈不是摘要，而是 FeedbackActionTable。
	9.	引用核验不是警告，而是 CitationCheckPanel。
	10.	知识胶囊不是收藏，而是 ReusableCapsuleCard。

4.6 对话页右侧上下文面板

右侧面板必须随对话上下文变化。

当用户做 PPT：显示老师要求、资料、页级大纲、三色溯源。
当用户读论文：显示论文列表、文献矩阵、引用状态。
当用户复习：显示考试倒计时、知识点地图、错题归因。
当用户做实验：显示实验日志、参数、数据文件、异常记录。
当用户处理反馈：显示反馈列表、对应位置、整改状态。

右侧面板不是固定资料栏，而是“当前任务上下文”。

4.7 对话页左侧任务流

左侧不是普通历史聊天，而是任务流。

每条记录显示：
	1.	任务标题。
	2.	当前状态。
	3.	最后一次 AI 动作。
	4.	是否待用户确认。
	5.	截止风险。
	6.	是否有后台任务运行。

示例：

“机器学习 PPT · 待选题 · AI 已准备 3 个方向”
“统计学期末 · 今日待复习 · 9 天后考试”
“组会汇报 · 核验中 · 2 条引用待确认”

4.8 底部命令输入栏

输入栏必须具有任务感，而不是空白聊天框。

组成：
	1.	自然语言输入。
	2.	上传按钮。
	3.	语音按钮。
	4.	当前模式标签：深度思考、联网检索、引用溯源、本地优先、轻量处理。
	5.	快捷命令。

快捷命令根据上下文变化：

PPT 项目：继续生成下一页、改这页、生成讲稿、换风格、查引用。
论文项目：生成矩阵、找研究空白、生成组会大纲、核验引用。
复习项目：开始今日题目、重排计划、解释错题、生成变式题。
实验项目：记录实验、分析异常、生成下次计划、整理组会汇报。
反馈项目：拆解反馈、标记已处理、生成回复信、查看版本差异。

4.9 页面状态示例

场景：本科 PPT
用户回到电脑端，打开 AI 对话控制台。

中央第一张卡：

“我已经解析了你上传的 6 个文件。这是一个 10 分钟课程 Presentation。老师要求覆盖背景、方法、应用和评价。我为你准备了 3 个选题方向。”

下面出现 3 张选题卡。用户选第 2 张后，AI 立刻显示：

“已选择方向 2。下一步我会生成 9 页 PPT 大纲。需要你确认后，我再进入逐页共创。”

随后出现 Skill 调用卡：

“调用 PPT Skill：生成页级大纲。”

大纲生成后出现 Artifact 卡：

“PPT 大纲已生成，进入 PPT Studio 逐页确认。”

场景：期末复习
AI 主动推送：

“统计学考试还有 9 天。你昨天错题集中在假设检验和置信区间。我已把今天计划重排为 2 个知识点 + 12 道变式题。”

卡片按钮：开始今日复习、查看错题归因、调整今日可用时间。

场景：导师反馈
用户粘贴导师反馈后，系统显示：

“我拆解出 7 条整改项，其中 3 条涉及方法部分，2 条涉及图表表达，2 条需要补充实验依据。”

下方是反馈整改表，右侧定位到论文或 PPT 对应位置。

5. 核心页面一：今日指挥中心 Today Command Center

4.1 页面目标

这个页面是知序的“驾驶舱”，但不能像 BI 大屏。它的核心不是展示数量，而是推动用户处理关键确认。

4.2 页面结构

顶部主区域：今日 Agent Brief。

示例：

“我已经处理完你昨晚上传的 9 个文件。机器学习 Presentation 可以开始选题；统计学期末复习计划已根据考试倒计时重排；导师反馈中还有 3 条整改未处理。”

下方三列：

第一列：待你确认。
	1.	PPT 选题。
	2.	报告大纲。
	3.	引用核验。
	4.	三方案选择。
	5.	导师反馈处理。
	6.	小组贡献报告确认。

第二列：AI 正在处理。
	1.	文件解析。
	2.	文献矩阵。
	3.	PPT 页面草稿。
	4.	错题归因。
	5.	实验日志整理。
	6.	引用交叉核验。

第三列：今天要做。
	1.	阅读 2 篇论文。
	2.	完成第 3 章复习。
	3.	确认 PPT 第 1–3 页。
	4.	处理导师反馈第 4 条。
	5.	晚上 9 点模拟答辩。

底部横向区域：风险与复用。
	1.	截止风险：报告还有 36 小时。
	2.	合规风险：有 5 个灰色内容未处理。
	3.	复用建议：发现 2 个历史知识胶囊可用于当前任务。
	4.	隐私提醒：某项目含未发表实验数据，建议本地优先。

4.3 视觉重点

今日页主视觉必须是“Agent Brief + 待确认卡片”。不要把资料总数、任务总数、知识图谱放在主视觉。

12. 多端体验策略

12.0 所有端首页统一为 AI 对话页

Web、手机、平板、PWA/桌面端进入产品后的默认首页都必须是 AI 对话页。差异只体现在布局和能力密度。

各端 AI 首页定义：
	1.	Web：AI Conversation Console。三栏结构，适合深度推进和共创。
	2.	手机：AI Capture Conversation。单栏结构，适合捕获任务、轻确认、查看推送。
	3.	平板：AI Reading & Co-creation Conversation。双栏结构，适合阅读、批注、复习和审阅。
	4.	PWA/桌面：AI Resident Console。强调后台处理、离线缓存、文件拖拽、多窗口产物编辑。

统一首页必须具备：
	1.	AI Brief：当前 AI 对用户任务的简报。
	2.	输入入口：文字、语音、上传、截图。
	3.	待确认卡：需要用户选择或确认的关键节点。
	4.	后台状态：AI 正在执行的任务。
	5.	主动推送：截止风险、任务停滞、计划重排、复用建议。
	6.	快捷动作：继续推进、上传资料、安排今天、进入 Canvas。

其他页面都不能作为默认首页，只能从 AI 对话首页进入。

12.1 手机端：AI 捕获对话与轻确认

手机端的核心不是编辑，而是把现实世界的学习科研任务捕获进系统。

手机端默认首页是 AI 捕获对话页，不是“今日”页、任务页或功能宫格。

底部导航可以保留，但默认落点必须是“AI”。建议底部导航：AI、任务、资料、胶囊、我的。

核心场景：
	1.	上课时拍老师 PPT。
	2.	语音记录老师作业要求。
	3.	上传文件。
	4.	查看 AI 是否处理完。
	5.	选择选题。
	6.	接收截止提醒。
	7.	确认今日计划。
	8.	做轻量题库。

手机端必须避免：复杂 PPT 编辑、复杂知识图谱、密集表格、重型项目管理。

12.2 Web 端：深度共创

Web 端默认首页是 AI Conversation Console。Web 端承担：选题、大纲、PPT、报告、论文矩阵、引用核验、导出、复盘。所有这些能力优先从 AI 对话中显式展开。

12.3 平板端：阅读与批注

平板端默认首页是 AI 阅读/共创对话页。平板端适合：论文精读、课堂纪要、PPT 审阅、复习刷题、导师反馈批注。

平板必须支持：手写批注转任务、图片批注转反馈、分屏阅读和对话。

12.4 桌面/PWA：长期驻留与离线

桌面/PWA 默认首页是 AI Resident Console。桌面/PWA 负责：本地缓存、文件拖拽、多窗口编辑、弱网支持、系统通知、离线任务。

?

13. 典型全链路体验

13.1 本科生 PPT

手机端捕获：老师布置任务，用户上传资料和语音。
后台：AI 解析文件、识别交付物、准备选题。
Web 今日页：显示“PPT 选题待确认”。
项目空间：用户选择选题。
Artifact Studio：逐页确认 PPT、生成讲稿、模拟答辩。
合规中心：检查三色溯源和引用。
知识空间：生成课程汇报胶囊。

13.2 期末复习

用户上传课件和考试日期。
AI 生成知识图谱和三套复习方案。
用户选择稳妥/加急方案。
手机每天提醒复习任务。
Exam Studio 做题并归因错题。
考前生成冲刺清单。
期末后沉淀课程知识胶囊。

13.3 硕博组会

用户上传论文、实验日志和导师反馈。
AI 生成文献矩阵、实验进展、问题清单。
用户选择组会汇报角度。
PPT Studio 生成组会 PPT。
答辩模拟生成导师可能追问。
会后上传反馈，系统生成整改任务和导师偏好。

13.4 论文返修

用户上传审稿意见和论文。
系统拆分审稿意见，绑定论文段落。
生成逐条整改计划和回复信草稿。
Document Studio 修改正文。
版本对比展示每条意见对应修改。
合规中心检查引用和结论。
归档为返修知识胶囊。

15. 视觉与交互方向

15.1 视觉气质

知序应该像“学术版 Linear + Artifact Canvas + AI Task Copilot”，而不是 Notion 克隆、ChatGPT 克隆、项目管理后台或科研 BI 大屏。

关键词：
	1.	高级但不装。
	2.	轻量但不玩具。
	3.	学术但不古板。
	4.	智能但不赛博。
	5.	有序但不企业 OA。

15.2 视觉层级

最重要的视觉层级应该是：
	1.	AI Brief。
	2.	待确认动作。
	3.	当前 Artifact。
	4.	证据与风险。
	5.	辅助数据。

统计数字、知识图谱、资料总数只能作为辅助层级。

15.3 动效原则

动效用于表达 Agent 正在工作、任务正在推进、证据正在核验、产物正在生成。

不能做纯炫技动画。
不能做电路、芯片、赛博粒子。
应使用低噪声、低频率、可解释的状态动效。

16. 不应该出现的错误设计
	1.	首页放大量项目统计卡。
	2.	中心放知识图谱装饰。
	3.	把对话作为边缘功能。
	4.	把产物编辑藏得太深。
	5.	用户打开项目后不知道下一步。
	6.	AI 后台处理没有进度反馈。
	7.	移动端复制 Web 复杂界面。
	8.	三色溯源只作为装饰标签。
	9.	合规风险只弹窗警告，不提供处理动作。
	10.	一键生成完整 PPT，然后让用户自己重改。
	11.	把 Skills 做成用户自己逛市场，而不是 Agent 自动调度。
	12.	把知识胶囊做成普通收藏夹。

?

17. 最终前端定义

知序的前端不是一个页面，也不是一个聊天框。它是一个多端连续的学习科研任务操作系统。

手机负责捕获现实任务。
Web 负责深度共创和核验。
平板负责阅读、批注和复习。
PWA/桌面负责离线、文件和长期驻留。
知识空间负责长期复用。
合规中心负责可信与边界。
Skills 负责能力扩展，但由 Agent 调度。

用户理想感知应该是：

“我把任务交给知序，它会先理解、先准备、先拆解；等我回来时，它不是给我一个粗糙成品，而是给我几个可选方向、明确下一步和可编辑产物。我参与关键判断，它处理繁琐执行；最终所有过程、证据和经验都会沉淀下来。”

这才是知序的前端设计基准。

18. 页面级排布与视觉样式总规范

18.1 总体视觉方向

知序的界面应呈现“学术可信 + AI 工作台 + 高级生产力工具”的气质。它不能像企业 OA，也不能像普通项目管理软件，更不能像传统教育 App。页面应该让用户感受到：系统正在理解任务、处理资料、调用能力、生成产物、等待确认、持续推进。

视觉关键词：
	1.	纸感：背景偏暖白，适合长时间阅读和写作。
	2.	秩序：清晰分栏、明确层级、少噪声。
	3.	可信：证据、引用、确认、风险状态要稳定、清楚。
	4.	智能：AI 正在工作需要有状态反馈，但不做炫技动画。
	5.	高级：少渐变、少装饰、强留白、强排版。
	6.	可操作：所有 AI 结果必须变成按钮、卡片、表格、Canvas 或确认项。

18.2 全局网格

Web 桌面端建议采用 12 栅格，最大内容宽度 1440–1560px。核心工作台允许全宽。

常用布局：
	1.	三栏工作台：左 280px，中间自适应，右 360px。
	2.	对话主工作台：左 300px，中间 1fr，右 380px。
	3.	Artifact Studio：左 320px，中间 1fr，右 380px。
	4.	大 Canvas 模式：左 80px 折叠栏，中间 1fr，右 420px。
	5.	移动端：单栏卡片流。
	6.	平板横屏：双栏 36% / 64%。

基础间距：
	1.	页面外边距：Web 24–32px，移动端 16px。
	2.	卡片内边距：20–28px。
	3.	面板间距：12–16px。
	4.	大模块间距：32–48px。
	5.	卡片圆角：20–28px。
	6.	主 Canvas 圆角：28–36px。

18.3 色彩系统

背景层：
	1.	App 背景：#F7F3EA，暖纸白。
	2.	卡片背景：#FBF8F1，轻纸白。
	3.	浮层背景：#FFFFFF / 88% 透明度，可加 backdrop blur。
	4.	暗色工作区：#101722，用于重点 AI 工作状态、合规层和品牌强调。

文本层：
	1.	主标题：#101722。
	2.	正文：#3F4650。
	3.	次级文字：#6A717C。
	4.	弱提示：#9AA1AA。

功能色：
	1.	证据绿：#7E9488，代表有来源、已核验、通过。
	2.	核验黄：#C6A25A，代表待确认、需要人工判断。
	3.	参考灰：#9DA3AD，代表 AI 推断、仅供参考。
	4.	风险红：#B65B5B，仅用于高风险、失败、删除确认。
	5.	深墨蓝：#101722，主按钮、品牌重点、深色面板。

三色溯源必须保持语义稳定：绿色永远代表可溯源，黄色永远代表待核验，灰色永远代表供参考，不能在其他语境随意复用。

18.4 字体与排版

界面字体：中文使用现代无衬线，英文与数字使用 Inter / SF Pro。品牌大标题可使用衬线字体或高质量宋体，营造学术气质。

字号建议：
	1.	页面巨标题：64–96px，适用于官网和空状态，不用于工作台密集区。
	2.	页面标题：32–44px。
	3.	模块标题：20–24px。
	4.	卡片标题：16–18px。
	5.	正文：14–16px。
	6.	标签与状态：12–13px。
	7.	表格内容：13–14px。

行高：长文阅读 1.7–1.85，工作台 UI 1.45–1.6。

19. 页面一：AI 对话控制台

19.1 页面定位

AI 对话控制台是知序的主入口，也是所有功能的显式操作面。用户打开产品后，不应先看到项目统计，而应看到 AI 对所有任务的工作状态：哪些任务已处理、哪些任务待确认、哪些任务需要用户执行、哪些风险需要处理。

19.2 桌面端排布

整体为四区结构：

左侧任务流，宽度 300px。
中间 AI 对话工作流，自适应宽度。
右侧上下文面板，宽度 380px。
底部命令输入栏，固定在中间区域底部。

左侧任务流
顶部为知序文字 Logo、全局新建按钮和搜索入口。

下方是任务分组：
	1.	待你确认。
	2.	AI 正在处理。
	3.	今天要做。
	4.	即将截止。
	5.	已归档。

每个任务卡高度 72–96px，显示：标题、当前状态、最后 AI 动作、截止倒计时和风险色点。

视觉样式：
	1.	默认卡片为透明或浅纸白。
	2.	当前选中任务用深墨蓝左边线 + 轻背景。
	3.	待确认任务显示核验黄小点。
	4.	AI 正在处理显示细进度条。
	5.	高风险任务显示低饱和红色边框，但不要大面积红色。

中央 AI 对话工作流
顶部是项目/任务状态条：
	1.	当前任务标题。
	2.	状态：待选题 / 生成中 / 核验中 / 待导出。
	3.	截止时间。
	4.	隐私模式。
	5.	额度状态。

中间是对话流，但对话流不是纯气泡，而是“消息 + 工作卡片”混排。

消息类型排布：
	1.	用户输入：右侧气泡，宽度不超过 70%。
	2.	AI Brief：大卡片，宽度 100%，放在关键节点。
	3.	Skill 调用卡：横向紧凑卡，显示工具名、进度、权限。
	4.	选题/方案卡：网格卡片，2–3 列。
	5.	Artifact 预览卡：大卡片，带进入 Canvas 按钮。
	6.	Human Gate：明显但克制的确认卡，按钮明确。
	7.	风险卡：带可处理动作，不只警告。

中央区视觉重点：

第一屏必须出现 AI Brief。例如：

“我已解析 6 个文件，识别到这是一个 10 分钟课程 Presentation。我准备了 3 个选题，建议先选方向。”

AI Brief 使用深墨蓝文字、浅纸白大卡、左侧细状态条。

右侧上下文面板
右侧不是固定资料栏，而是随任务切换。

PPT 任务显示：老师要求、资料、页级大纲、三色溯源。
论文任务显示：论文列表、文献矩阵、引用状态。
复习任务显示：考试倒计时、今日计划、错题归因。
实验任务显示：实验日志、参数、异常记录。
反馈任务显示：反馈条目、绑定位置、整改状态。

右侧面板采用 Tabs：上下文、资料、证据、活动。默认显示“上下文”。

底部命令输入栏
输入栏固定在中央底部，不横跨全屏。

结构：
	1.	左侧模式标签：当前是“PPT 共创 / 引用核验 / 本地优先 / 深度处理”。
	2.	中间自然语言输入框。
	3.	右侧上传、语音、发送按钮。
	4.	上方一排快捷命令 chips。

快捷命令根据任务变化。例如 PPT：继续生成下一页、改这一页、生成讲稿、查引用、进入 Canvas。

19.3 移动端排布

移动端 AI 对话控制台采用单栏。

顶部：当前任务状态 + 返回任务流。
中间：对话卡片流。
底部：输入栏 + 快捷命令。
右侧上下文面板变为底部抽屉。

移动端重点展示：AI Brief、待确认卡、今日任务、轻量选择。复杂 Canvas 不在移动端主做。

20. 页面二：今日指挥中心

20.1 页面定位

今日页是全局指挥中心，用于让用户快速知道“今天该做什么”。它可以被设计为 AI 对话控制台的首页态，也可以是独立页面。

20.2 桌面端排布

顶部大区：Agent Daily Brief。

大卡片高度 220–280px。左侧为今日总结，右侧为 AI 后台状态。

示例结构：

左侧：

“今天有 4 件事需要你确认。机器学习 PPT 已准备选题；统计学复习计划已重排；导师反馈还有 3 条未处理；组会汇报大纲已生成。”

右侧：
	1.	AI 已完成：12 个文件解析。
	2.	正在处理：2 个任务。
	3.	待确认：4 项。
	4.	风险：1 项。

中部三列：

第一列：待你确认。
第二列：AI 正在处理。
第三列：今天要做。

底部横向：风险与复用建议。

20.3 卡片样式

待确认卡：核验黄状态点 + 主按钮“去确认”。
AI 处理中卡：进度条 + 当前步骤。
今天要做卡：复选状态 + 预计耗时。
风险卡：轻红边框 + 处理按钮。
复用建议卡：证据绿 + “复用胶囊”按钮。

20.4 视觉层级

今日页的第一视觉是 Agent Brief，不是图表。图表只作为辅助，例如本周任务压力、考试倒计时、复习完成率。

21. 页面三：任务捕获页 Capture

21.1 页面定位

任务捕获页用于手机、Web、桌面端快速输入现实世界任务。它要像“把任务丢给知序”的入口，而不是复杂表单。

21.2 移动端排布

顶部：一句品牌型引导。

“把任务丢给知序。”

中间：大输入卡。

输入卡包含：
	1.	文本输入。
	2.	语音按钮。
	3.	拍照按钮。
	4.	文件按钮。
	5.	截图粘贴入口。

下方：快捷任务类型。
	1.	做 PPT。
	2.	写报告。
	3.	读论文。
	4.	准备考试。
	5.	整理实验。
	6.	导师反馈。
	7.	小组作业。
	8.	上传资料。

底部：已添加资料堆栈。

资料堆栈采用横向小卡，显示文件名、类型、解析状态。

21.3 Web 端排布

Web 捕获页可作为模态浮层或独立页。

模态宽度 720–860px，居中。

左侧是输入区，右侧是文件区。提交后直接进入 AI 理解中状态。

21.4 视觉样式

捕获页要轻、快、低压力。背景可为纸白，输入卡为白色半透明，按钮使用深墨蓝。

不要让用户填写一堆字段。字段由 AI 理解后反推确认。

22. 页面四：AI 预处理完成页 Ready Review

22.1 页面定位

这是知序最关键的“回宿舍打开电脑”页面。用户在手机端提交任务后，Web 端看到 AI 已经处理完，并准备好了下一步选择。

22.2 桌面端排布

左侧：任务摘要，宽 300px。

内容：任务类型、截止时间、交付物、资料、老师要求、处理状态。

中间：AI 准备结果，主视觉。

顶部为 AI Brief：

“我已经解析了 6 个文件，识别到这是一个 10 分钟 Presentation。建议做 8–10 页。我准备了 3 个选题方向。”

下方为候选方向卡片，3 张横排或 2+1 排布。

每张卡：
	1.	选题名。
	2.	一句话价值。
	3.	推荐指数。
	4.	资料覆盖率。
	5.	预计页数。
	6.	难度。
	7.	风险。
	8.	选择按钮。

右侧：风险与证据。

内容：资料覆盖、缺失信息、评分标准匹配、三色预估、时间风险。

22.3 视觉样式

候选方向卡片应该有明显层级。推荐选题可以用细金色边框和“推荐”标签，但不要夸张。卡片底部有“查看依据”，点击后右侧面板定位来源。

22.4 交互

用户选择方向后，中间区域平滑过渡到“大纲确认卡”。不要跳转到另一个冷冰冰页面。

23. 页面五：项目推进工作台

23.1 页面定位

项目推进工作台用于单个项目的持续推进，是 AI 对话控制台的项目态。

23.2 排布

左栏：状态轨。

状态轨垂直展示：捕获、理解、规划、准备、共创、核验、交付、归档。

每个状态显示完成情况、当前卡点、负责人。

中栏：对话与工作卡。

按时间顺序展示 AI 和用户交互，但关键内容必须以结构化卡片展示。

右栏：项目上下文。

Tabs：资料、证据、任务、活动、协作。

23.3 视觉样式

状态轨采用细线 + 小圆点，不做厚重流程图。当前状态使用深墨蓝，已完成使用证据绿，待确认使用核验黄。

对话区卡片采用纸白背景，大卡圆角，避免满屏气泡。

24. 页面六：Artifact Studio

24.1 页面定位

Artifact Studio 是产物深度共创页面，所有文档、PPT、论文矩阵、复习计划、题库、实验日志都应使用统一框架。

24.2 通用三栏排布

左侧 AI 控制台：320px。

内容：当前产物状态、AI 建议、对话命令、快捷操作、历史操作。

中间 Canvas：自适应。

内容：实际产物，如 PPT 页面、文档段落、题库、实验表格。

右侧 Inspector：380–420px。

内容：当前块目标、证据来源、三色溯源、质量问题、版本、评论。

24.3 PPT Studio 排布

左侧新增页面缩略图栏，宽 88–120px，可折叠。

中间为当前页大画布。画布比例 16:9，居中，周围留白。

右侧 Inspector 显示：
	1.	当前页目标。
	2.	页面大纲位置。
	3.	来源资料。
	4.	三色溯源。
	5.	文字密度。
	6.	视觉建议。
	7.	讲稿。
	8.	操作：重写本页、改短、加图示、生成讲稿、锁定本页。

24.4 Document Studio 排布

中间是文档页面，模拟纸张宽度 760–860px。左侧为 AI 写作控制台，右侧为段落 Inspector。

选中段落后出现悬浮工具条：改短、扩写、学术化、加引用、统一文风、转 PPT、标记待核查。

24.5 Literature Studio 排布

中间为文献矩阵表格和研究脉络。顶部可切换：矩阵、对比、综述、研究空白、组会角度。

右侧 Inspector 展示当前论文来源、引用状态和证据页码。

24.6 Exam Studio 排布

中间为今日复习任务和题目区。右侧显示错题归因、知识点地图、考试倒计时。左侧为复习计划和章节导航。

24.7 Experiment Studio 排布

中间为实验日志表单和结果记录。右侧显示参数版本、异常归因、对照实验建议。左侧为实验批次列表。

25. 页面七：知识空间 Knowledge OS

25.1 页面定位

知识空间不是资料库，而是长期资产系统。

25.2 排布

顶部：知识资产总览。
	1.	知识胶囊。
	2.	课程图谱。
	3.	文献矩阵。
	4.	实验日志。
	5.	错题归因。
	6.	导师偏好。
	7.	术语库。

中间：智能复用推荐。

卡片按“当前任务相关性”排序。每张卡说明：为什么推荐、可复用什么、会影响哪个当前任务。

右侧：资产维护。

合并、删除、共享、隐私、导出。

25.3 视觉样式

知识胶囊卡片可以更有品牌感，采用胶囊形态或圆角长卡。每张胶囊显示项目来源、时间、可复用内容、隐私范围。

26. 页面八：合规与溯源中心

26.1 页面定位

合规中心是处理风险的地方，不是吓用户的地方。

26.2 排布

顶部：合规总览。
	1.	绿色内容比例。
	2.	黄色待确认比例。
	3.	灰色供参考比例。
	4.	引用核验状态。
	5.	Human Gate 记录。

中间：风险清单。

按严重程度排序：必须处理、建议处理、可忽略。

右侧：预览与定位。

点击风险项后，右侧显示对应段落、PPT 页面或引用位置。

底部：导出合规报告。

26.3 视觉样式

合规中心可以使用更深的背景或更强的分隔，体现严肃性。但不能像安全警告页。所有风险项必须有处理按钮：补来源、确认、删除、改写、保留为备注。

27. 页面九：Skills 能力面板

27.1 页面定位

Skills 不应该是用户自己逛插件商店的主要入口，而是展示 Agent 调用了哪些能力、这些能力有什么权限和成本。

27.2 排布

顶部：当前项目正在使用的 Skills。

中间：官方 Skills 和推荐 Skills。

右侧：调用日志与权限记录。

Skill 卡片显示：名称、用途、触发条件、权限、风险、最近调用、启用状态。

27.3 视觉样式

Skill 卡片不要像 App Store。它更像“能力说明 + 权限凭证”。高风险 Skill 要有明显边界和按次确认。

28. 页面十：设置与隐私

28.1 页面定位

设置页重点是信任建立：隐私模式、数据同步、模型使用、记忆管理、Skill 权限、导出记录。

28.2 排布

左侧设置导航。右侧内容区。

核心设置：
	1.	隐私模式：云端、本地优先、组织隔离。
	2.	记忆管理：用户偏好、导师偏好、知识胶囊。
	3.	Skill 权限：已授权、待确认、已禁用。
	4.	数据管理：删除、导出、同步状态。
	5.	通知设置：强提醒、弱提醒、每日摘要、免打扰。
	6.	模型设置：速度优先、质量优先、本地优先。

28.3 视觉样式

设置页必须清楚、可信，不做营销化。敏感设置要给解释，不只给开关。

29. 多端适配细则

29.0 多端首页统一规定

所有端口的首页都是 AI 对话页。

不同端的 AI 首页差异：
	1.	Web 首页：三栏 AI 工作台，突出任务推进、工作卡片、Canvas 入口和证据面板。
	2.	手机首页：单栏 AI 对话流，突出快速捕获、轻确认、提醒和后台状态。
	3.	平板首页：双栏 AI 对话 + 阅读/批注画布，突出论文、课堂、复习和 PPT 审阅。
	4.	桌面/PWA 首页：AI 驻留控制台，突出离线状态、文件拖拽、后台队列和多窗口工作。

导航栏只能是辅助，不能替代 AI 首页。

29.1 手机端

手机端优先级：捕获 > 确认 > 提醒 > 轻复习 > 查看 AI 进度。

手机端首页结构：
	1.	顶部 AI Brief：今天有什么待确认、AI 处理完什么、有什么风险。
	2.	中央单栏对话流：用户任务输入、AI 理解卡、待确认卡、推送卡。
	3.	快捷捕获区：语音、拍照、文件、截图、文字。
	4.	后台状态条：正在解析、正在生成、正在核验。
	5.	底部输入栏：自然语言 + 上传 + 语音。

底部导航：AI、任务、资料、胶囊、我的。

主要页面使用卡片流，避免复杂分栏。

29.2 平板端

平板端优先级：阅读 > 批注 > 审阅 > 复习 > 轻编辑。

横屏使用双栏：左对话/大纲，右 Canvas。竖屏使用上下结构。

29.3 桌面/PWA

桌面端优先级：深度编辑、文件拖拽、多窗口、离线缓存、后台通知。

桌面可支持：一个窗口打开 AI Console，另一个窗口打开 Artifact Studio。

30. 页面设计验收标准

30.0 首页验收标准

所有端首页必须满足：
	1.	默认进入 AI 对话页。
	2.	不能默认进入 Dashboard、项目列表、资料库、知识图谱、功能宫格。
	3.	首页第一屏必须出现 AI Brief 或任务型输入入口。
	4.	首页必须显示至少一类显式 Agent 状态：待确认、AI 正在处理、今日建议、风险提醒、后台队列。
	5.	用户可以在首页直接上传资料、语音输入、继续推进任务。
	6.	其他空间必须能从 AI 对话页以卡片、抽屉、侧栏或 Canvas 的形式展开。
	7.	如果用户没有任务，首页也不能是空聊天框，而应显示场景化捕获入口：做 PPT、写报告、读论文、准备考试、整理实验、导师反馈。

所有页面设计必须满足以下验收：
	1.	用户打开页面 5 秒内知道下一步该做什么。
	2.	AI 后台正在做什么必须可见。
	3.	所有高风险内容必须可定位、可处理、可确认。
	4.	产物不能只是生成结果，必须可编辑、可核验、可版本对比。
	5.	手机端能快速捕获任务，Web 端能深度共创。
	6.	用户不需要自己找功能入口，AI 应在对话中显式调用能力。
	7.	三色溯源不能装饰化，必须成为可点击、可处理的界面对象。
	8.	知识胶囊不能只是收藏，必须能跨项目推荐和复用。
	9.	Skill 调用不能黑箱，必须显示用途、权限、状态和结果。
	10.	页面不能以项目统计为中心，必须以任务推进为中心。
