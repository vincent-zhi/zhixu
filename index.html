Project OS

资料进入知序后，不是被总结，而是被组织成可推进的项目。

这条链路来自 PRD 和技术规定：Source Pipeline、Task Breakdown、Planner、Dispatcher、Verifier、Watcher、Human Gate、Knowledge Capsule。页面用滚动叙事把它做成产品感，而不是功能清单。

Product capabilities

真正的特点，是把 AI 能力放进学习科研流程，而不是把流程压扁成一次生成。

01

项目仪表盘

任务进度、待办、资料、产物、最近 AI 建议和截止风险都挂靠 Project。Watcher 会发现停滞、提醒延误并重排计划。

  • 当前状态与下一步动作
  • 推荐 / 加急 / 稳妥三方案
  • 任务责任标签与截止风险
02

Artifact Canvas

报告、PPT、综述、复习计划和实验记录都以块级结构编辑，支持证据绑定、评论、版本 diff 与导出。

03

知序 Skills

PPT、Word、PDF、OCR、Literature、Citation、Exam Coach、Experiment Log 等能力被封装为可治理的 Skill Runtime。

04

导师反馈沉淀

导师批注、返修意见和组会纪要会被拆解为整改项，绑定章节、页面、实验任务或论文段落。

05

Agent OS

需求理解器、Planner、Dispatcher、Worker、Verifier、Watcher、Memory Manager、Reflection Engine 和 Human Gate 组成项目制 Agent OS。

Source · responsibility · risk

三色溯源不是装饰，是知序的可信边界。

绿色内容可追溯，黄色内容需要用户核验，灰色内容只能作为参考。知序不会承诺“100% 无幻觉”，而是把来源、责任和风险显示在最终交付之前。

绿色上传资料 / 用户输入 / 可核验公开来源
黄色AI 归纳、比较、改写，等待用户确认
灰色AI 推断、拓展建议、研究设想
导出前检查无来源引用：0待核验观点：7L3 风险请求：已拦截

Use cases

四类真实任务，展示知序为什么不是普通 AI 工具。

本科 Presentation

从老师资料到 10 分钟汇报

上传课件、作业要求和语音说明后，知序生成选题候选、页级大纲、讲稿、视觉规范与 PPTX 导出路径。

硕士组会

6 篇论文变成可讲的比较框架

提取研究问题、方法、实验、数据集、结论、局限和可复现性，形成论文矩阵和三种汇报路径。

期末复习

知识图谱连接计划、练习和错题

课程资料、作业、往年题和课堂笔记进入 CourseKnowledgeGraph，生成 14 天计划并动态重排。

科研实验

实验过程可追踪、可复盘

把数据准备、baseline 复现、参数记录、结果表格、可视化和组会汇报变成状态清楚的任务树。

Brand boundary

知序更可信、更可控、更会陪跑、更能沉淀。

它不替代老师或导师，不自动提交作业，不伪造引用或实验数据。它把复杂学习科研任务拆成可确认、可编辑、可追踪、可复用的项目。
