# 知序技术规定文档 v1.0

文档类型：技术规定 / Technical Specification  
适用范围：Web、iOS、Android、桌面端、后端服务、AI Agent、Skill Runtime、数据平台、组织版  
目标：将知序所有功能转化为工程可实现、可测试、可审计、可扩展的技术规定。

---

## 1. 技术总原则

知序的技术系统围绕“项目制学习科研 Agent OS”构建。系统最小业务单元不是聊天会话，而是 Project。所有资料、任务、对话、计划、产物、证据、版本、权限、日志、提醒、知识胶囊、导师反馈、Skill 调用都必须挂靠到 Project 或从 Project 派生的对象上。

所有 AI 输出必须经过三类约束：来源约束、责任约束、风险约束。来源约束回答“内容从哪里来”；责任约束回答“谁对内容负责”；风险约束回答“是否能进入最终交付”。

高风险操作必须进入 Human Gate，包括最终导出、对外发送、删除文件、覆盖版本、引用补全、实验结论生成、研究贡献生成、调用第三方 Skill、访问外部账号、使用付费服务、处理敏感资料上云。

---

## 2. 总体架构

系统采用“客户端多端 + API Gateway + 业务服务 + Agent OS + Skill Runtime + 数据层 + 可观测平台”的架构。

Client Layer：Web、iOS、Android、Desktop/PWA。  
API Gateway：统一鉴权、限流、路由、审计、版本兼容。  
Business Services：用户、项目、任务、资料、产物、日程、协作、订阅、通知、组织管理。  
Agent OS：需求理解器、Planner、Dispatcher、Worker、Verifier、Watcher、Memory Manager、Reflection Engine、Human Gate。  
Skill Runtime：官方 Skills、认证第三方 Skills、沙箱、权限控制、调用日志、版本管理。  
Data Layer：关系数据库、对象存储、向量库、全文检索、事件流、缓存、审计日志。  
AI Infra：模型网关、RAG 管线、文档解析、OCR、引用核验、评测系统、成本控制。  
Observability：Agent Trace、任务队列监控、成本监控、质量评估、错误告警、安全审计。

推荐技术栈：Next.js + React + TypeScript、React Native/Flutter、PostgreSQL、Redis、S3、OpenSearch、pgvector/Milvus/Qdrant、Temporal、Model Gateway。

## 3. 部署形态

云端模式：普通用户默认模式，云端完成解析、索引、生成、导出和多端同步。

本地优先模式：敏感项目可选择，本地客户端负责文件解析、轻量索引、基础问答、任务规划和离线缓存。复杂生成前提示用户将哪些内容发送到云端。

私有/组织模式：面向高校、实验室、研究机构、企业研发团队，支持独立对象存储、独立向量库、组织级权限策略、审计日志和模型策略。

## 5. Agent OS 技术规定

需求理解器负责从自然语言、语音、图片、文件、日程、聊天记录中抽取目标、交付物、截止日期、资料范围、风险、缺失信息和敏感信息。

Planner 负责生成任务树、依赖关系、三方案规划、风险评估、Human Gate 节点和 Skill 调用候选。

Dispatcher 负责任务分发，决定任务由模型、Skill、本地服务、云端服务还是用户完成。

Worker 负责执行子任务，包括摘要、提取、生成、改写、题库、PPT 页面、实验日志整理。

Verifier 负责独立校验 Worker 输出，至少检查事实、引用、三色责任、格式、逻辑、合规和导出完整性。

Watcher 负责状态追踪、任务停滞识别、提醒、重排计划和后台任务完成通知。

Memory Manager 负责用户偏好、项目记忆、知识胶囊、导师偏好、术语库和跨项目推荐。

Reflection Engine 负责任务复盘、缺陷归因、知识胶囊生成和下次任务建议。

Human Gate 负责高风险节点确认和审计记录。

所有 Agent 输出必须包含 output_type、structured_result、confidence、required_confirmations、evidence_refs、risk_flags、next_actions、cost_estimate。

## 6. Skill Runtime 技术规定

Skill 类型：Native Skill、Workflow Skill、Sandbox Script Skill、External API Skill、Local-only Skill。

默认禁止权限：读取全部文件、读取剪贴板、访问 Cookie/密钥、删除文件、外发文件、执行任意命令、调用付费 API、访问其他项目。

按需授权权限：读取当前项目指定文件、读取当前项目索引、写入当前项目产物、访问指定公开网页、读取指定 GitHub 仓库、创建项目提醒。

高风险黑名单：自动提交作业、自动投稿、自动发送邮件、自动下载受限数据库全文、执行未经审查代码、上传未发表资料到第三方 Skill。

Skill 调用生命周期：触发判断 → 权限检查 → 配额检查 → 沙箱准备 → 输入封装 → 执行 → 输出结构化 → Verifier 检查 → 写入草稿/结果 → 记录日志。

官方 Skills：Task Breakdown、Progress Tracking、PPT、Word、PDF、OCR、Excel/Data、Literature、Citation、Calendar、Exam Coach、Presentation Coach、Experiment Log、Image、Diagram、GitHub、Web Research、Translation、Style Unifier、Knowledge Capsule、Mentor Feedback、Review Response、Grant Application、Academic Resume、Group Collaboration、Plagiarism Risk Precheck、Offline Cache、Termbase。

## 7. 文件与资料解析

Source Pipeline：上传校验 → 安全扫描 → 类型识别 → 敏感性检测 → 文本抽取 → OCR → 版面识别 → 表格抽取 → 图片区域提取 → 元数据抽取 → 摘要生成 → 索引构建 → 证据定位。

PDF 保留页码、段落、表格、图片区域、参考文献区。PPTX 解析页标题、层级、备注、图表、图片、布局。DOCX 解析标题层级、段落、表格、引用、批注和修订痕迹。图片进入 OCR、版面区域、手写识别和公式识别。表格进入字段类型推断、缺失值统计和图表建议。

敏感资料包括未发表论文、实验数据、个人身份资料、导师反馈、课程内部资料、成绩单、简历、保密项目关键词。敏感资料触发处理模式提示。

## 8. 后台任务与成本控制

后台任务类型：parse_source、build_index、generate_summary、detect_task、generate_plan、generate_outline、generate_questions、verify_citations、export_artifact、create_capsule、sync_offline_changes。

队列优先级：用户当前等待 > 即将截止任务 > 导出任务 > 预处理任务 > 批量任务 > 周期性任务。

Cost Controller 在任务执行前计算预算。超过额度时必须提示降级方案：轻解析、只解析选中文件、延后处理、升级套餐、手动上传摘要。

---

## 9. 任务状态机

状态：Captured、Understanding、Planned、Preparing、Waiting User、Executing、Verifying、Ready to Deliver、Tracking、Completed、Archived、Risk、Failed。

每个状态必须有 entry_condition、exit_condition、owner、allowed_actions、timeout_policy、risk_policy。

Watcher 定时扫描 due_date、last_user_activity_at、completion_rate、pending_confirmations、risk_flags、quota_status，生成提醒、补救计划、任务重排、资料缺口提示和导出风险提示。

## 10. Artifact Canvas

Artifact Canvas 是长文档、PPT 大纲、复习计划、实验记录、导师反馈整改、知识胶囊的统一编辑工作台。

前端结构：左侧项目对话和 AI 命令，中间结构化文档编辑器，右侧大纲、证据、评论、版本、风险面板，顶部分享/导出/版本/合规检查，底部局部命令输入。

编辑器必须支持 block_id，所有编辑操作产生 operation log，支持撤销、重做、版本对比、评论、证据绑定。

Streaming Document 必须可暂停。用户在生成中修改方向时，系统必须保留已生成内容，并创建新分支。

## 11. 功能技术规定摘要

任务创建支持文本、语音、图片、文件、网页链接、日历事件、粘贴消息、邮件转发内容、聊天记录导入。

PPT 共创使用 SlideBlock，包含 slide_id、title、objective、layout_type、content_blocks、visual_suggestion、speaker_notes、evidence_refs、responsibility_color、status、version_id。

Word/报告使用 heading、paragraph、table、figure、citation、formula、checklist 块级结构。段落级 AI 命令包括改短、扩写、学术化、加例子、加引用、降重、转 PPT 页、生成讲稿。

论文阅读输出 PaperMatrix，字段包括 problem、method、data、metrics、main_results、limitations、future_work、relevance_to_project。

考试复习构建 CourseKnowledgeGraph，节点包括 chapter、concept、formula、example、assignment、mistake、question_type，边包括 prerequisite、similar、often_confused、appears_in、tested_by。

知识胶囊生成流程：收集项目摘要 → 提取可复用结构 → 提取确认观点 → 提取任务计划 → 提取错题/实验/反馈 → 生成胶囊草稿 → 用户确认保存。

导师反馈输入后拆解为整改项，绑定 ArtifactBlock，支持状态追踪和版本对比。

## 12. 合规与引用核验

风险等级：L0 安全、L1 可辅助、L2 需确认、L3 拒绝或改写为安全帮助。

L3 包括考试实时作答、伪造实验数据、伪造引用、代写整篇提交稿、规避查重、越权下载数据库文献、自动提交作业。

引用核验检查正文引用、参考文献列表、论述观点、来源证据一致性。模型生成的文献信息必须进入 CitationRecord，未经核验不得进入最终参考文献列表。

全流程 AI 学术溯源报告包含项目、产物、AI 使用范围、人工修改记录、引用核验状态、三色比例、高风险内容处理记录。报告只作为辅助自查材料，不宣称官方认可。

## 13. 多端同步与离线

采用块级同步。每个块记录 version_id、updated_at、device_id、updated_by。

离线允许查看资料、编辑大纲、做题、改任务、查看错题、写笔记。离线操作写入本地 operation queue。

同一 block 被不同端修改时进入冲突面板，展示本地版本、远端版本、建议合并版本。不得静默覆盖。

缓存必须加密，用户可清理本地缓存，敏感项目缓存遵守隐私模式。

---

## 14. 数据安全

文件传输使用 TLS。对象存储服务端加密。敏感字段单独加密。组织版支持自带密钥或专属密钥。

用户可删除项目、资料、产物、记忆、知识胶囊。彻底删除后异步清理对象存储、向量索引、全文索引和缓存。

默认不使用用户文件训练公共模型。第三方模型调用必须在隐私政策中说明。敏感资料上云必须有确认记录。

## 16. 质量评测

AI 质量评测维度：事实准确性、引用准确性、证据覆盖率、文风一致性、任务完成度、幻觉率、用户采纳率。

文献核验评测集：真实引用、虚假 DOI、错位引用、残缺引用、重复引用、格式错误。

导出质量评测：DOCX/PPTX/PDF 的结构、样式、页码、图表、引用、中文字体、公式、目录。

安全评测：越权文件读取、第三方 Skill 权限绕过、提示词注入、敏感资料外发、自动提交行为。

---

## 17. 绝对禁止行为

系统不得伪造文献、伪造实验数据、代替用户考试实时作答、自动提交作业、自动投稿、自动发送导师邮件、绕过数据库版权限制下载文献、无确认上传敏感资料到第三方、静默覆盖用户内容、删除用户文件不留确认记录。

系统不得对外承诺 100% 无幻觉、官方认可、零风险投稿、完全替代老师或导师。

---

## 18. 最终技术定义

知序的技术核心不是单个大模型，也不是单个文档生成器，而是项目制 Agent OS。它以 Project 为中心，以 Artifact 为交付形态，以 Evidence 为可信基础，以 Task State Machine 为推进机制，以 Skill Runtime 为执行层，以 Human Gate 为合规边界，以 Knowledge Capsule 为长期资产沉淀。
